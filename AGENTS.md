## Project Overview

An RAG application built with **LangChain / LangGraph**, exposing RAG functionality via a **FastAPI** REST API. Persists data and vectors in **Supabase** (PostgreSQL + pgvector). Managed entirely with **uv**.

---

## Tech Stack

| Layer | Library / Tool |
|---|---|
| RAG framework | `langgraph`, `langchain` |
| API server | `fastapi`, `uvicorn` |
| Data validation | `pydantic` v2 |
| Database / vector store | `supabase-py`, `pgvector` |
| Testing | `pytest`, `pytest-asyncio` |
| Linting / formatting | `ruff` |
| Package manager | `uv` |
| Env vars | `python-dotenv` |

## Code Conventions

### General
- Python **3.12+** minimum.
- **Async-first where applicable** ŌĆö use `async def` for all FastAPI route handlers and any I/O-bound functions (DB calls, LLM calls).
- **Strict type hints** on every function signature, including return types. No bare `Any` unless unavoidable.
- **Docstrings on every function and class** using Google-style format.

### LangGraph
- **`graph.py` is assembly only** ŌĆö it imports nodes and wires them together. No business logic here.
- **Each node lives in its own file** under `src/agent/nodes/`. Node functions are `async def`.
- `AgentState` is a `TypedDict` defined in `src/agent/state.py`.
- Use `Annotated` fields with `operator.add` for list fields that accumulate across steps.

### Supabase / Database
- All Supabase client access goes through `src/db/client.py` ŌĆö never instantiate the client inline.
- Database type definitions are autogenerated in `src/db/database_types.py`. DO NOT EDIT THIS FILE.
- Database schema is autogenerated in `src/db/schema.sql`. DO NOT EDIT THIS FILE.
- Use parameterised queries; never interpolate user input into SQL strings.

### Environment Variables
- All secrets and configuration come from `.env` via `python-dotenv`.
- Access config only through the `Settings` object in `src/config.py` (Pydantic `BaseSettings`).
- Never hardcode secrets, API keys, or connection strings.

## Testing

- Test runner: `uv run pytest`
- Always follow RedŌĆōGreenŌĆōRefactor TDD
- Use `pytest-asyncio` with `asyncio_mode = "auto"` in `pyproject.toml`.
- **Tests are co-located with their target** ŌĆö a test for `nodes/retriever.py` lives at `nodes/retriever_test.py`, not in a separate `tests/` directory.
- Test files are named `<module>_test.py` and live in the same directory as the module they test.
- Mock all external services (Supabase, OpenAI) in unit tests ŌĆö never hit live APIs in tests.
- Integration tests that require a live DB are marked `@pytest.mark.integration` and skipped by default.

## What NOT to Do

- Do not commit `.env` (it is in `.gitignore`)
- Do not add business logic to `graph.py` ŌĆö assembly only
- Do not instantiate the Supabase client outside `src/db/client.py`
- Do not use `pip install` ŌĆö always use `uv add`
- Do not use bare `except:` ŌĆö always catch specific exceptions
